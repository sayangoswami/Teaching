ezui: The Modern Java UI Toolkit

🟦 SECTION 1: THE STRUCTURE

What is ezui?

ezui is a beginner-friendly Java library designed to bridge the gap between “Learning Java” and “Building Apps.”

  • Why use it? No messy LayoutManagers, built-in navigation, and modern styling out of the box.

  • The Package: Always start with import ezui.*;

Big-Picture Architecture

  • Window: The “House” (Main Frame + Sidebar).

  • Page: A “Room” (One specific screen).

  • Grid: The “Furniture Layout” (Rows and Columns).

  • Component: The “Furniture” (Buttons, Text, etc.).

Window: The App Shell

The House for Your Pages

The Window is the “Master Container.” It creates a modern, split-pane interface with a Navigation Sidebar on the left and a Scrollable Content Area on the right.

The 4-Step Setup

To launch your application, follow this exact sequence in your Main.java::: {.fragment} The program wont compile yet, because we haven’t defined the Login and Dashboard pages, but this is the skeleton you will build on top of.


// 1. Set the Style (Do this FIRST!)
Theme.initialize("Light", "Modern");

// 2. Create the Window (Title, Width, Height)
Window app = new Window("Student Portal", 1000, 700);

// 3. Register your Pages 
// NOTE: The FIRST page you add will be the one displayed at startup!
app.addPage(new LoginPage()); 
app.addPage(new DashboardPage());

// 4. Launch!
app.show();

Tips:

  • Sidebar Magic: Every time you call addPage(), a button with that Page’s name is automatically created in the sidebar. Clicking it switches the screen instantly.
  • Smart Sizing: Use 1000x700 as a safe default. It fits perfectly on most laptops without hiding the taskbar.
  • The “Show” Rule: Always call app.show() or app.setVisible(true) at the very end of your main method to ensure all your components are properly sized and ready to view.

Page — Building Your Screen

The Blueprint of a Screen

Every screen in your app (like Login, Dashboard, or Settings) must extend a Page. Think of a Page as a named container that holds a Grid.

The Page Constructor

When you create a Page, you must tell ezui three things: its Name, how many Rows it has, and how many Columns it has.

public class DashboardPage extends Page {
    public DashboardPage() {
        // super(Name, Rows, Columns, ColumnStyles...)
        super("Dashboard", 3, 2, ColumnStyle.FIXED(), ColumnStyle.GROW());
    }

    @Override
    public void onCreate() {
        // This is where you add your components!
    }
}

Understanding the Arguments:

  1. Name ("Dashboard"): This is the text that will appear on the button in the Sidebar.
  2. Rows (3): How many horizontal “slots” your page has from top to bottom.
  3. Columns (2): How many vertical “slots” your page has from left to right.
  4. ColumnStyles: These tell the columns how to behave.
    • ColumnStyle.FIXED(): Stays as small as possible (good for labels/icons).
    • ColumnStyle.GROW(): Stretches to fill all remaining screen space (good for Tables/Charts).

Tips:

  • The Grid is Your Canvas: You can’t add a component to a Page without a Grid slot. If you need more space, increase the Row/Column numbers in the constructor.
  • Infinite Height: Don’t worry about making the page too long! ezui automatically adds a Scrollbar if your content exceeds the window height.
  • The onCreate Rule: Always put your root.add() code inside onCreate. If you put it in the constructor, the Window might not be ready to display it yet!

Grid: The Layout Engine

Everything inside a Page lives in a Grid.

The “Map” of Your Screen

Think of a Grid as a spreadsheet. Every “cell” has a unique address: (row, column). In programming, we always start counting at 0.

How to Add Components You use the root.add() method inside your Page’s onCreate() function.

// 1. Simple placement (Fills the column, aligns to top)
root.add(new Text("Header"), 0, 0);

// 2. Precise placement (Choose your alignment)
root.add(new Button("Save"), 1, 1, Alignment.RIGHT);

The Alignment Rulebook:

  • Alignment.LEFT: Sticks the component to the left side of the cell.
  • Alignment.CENTER: Places the component right in the middle.
  • Alignment.RIGHT: Sticks the component to the right side.
  • Alignment.FILL: Stretches the component to be as wide as the entire column (Perfect for TextBoxes and Buttons!).

Tips:

  • Overlapping: You cannot put two components in the same (row, column) address. The second one will simply replace the first!
  • Empty Spaces: You don’t have to fill every cell. If you leave (0, 1) empty, it will just show as a blank space in your layout.
  • Pro-Tip: Use Alignment.FILL for your input fields (TextBox) so they look consistent and professional across the page.

The onCreate Master Example

Putting the Grid to Work

Inside your Page class, the onCreate() method is your construction site. Here is how you would build a simple “Profile” layout using the Grid coordinates we just learned.

@Override
public void onCreate() {
    // Row 0: A big heading that spans the top
    root.add(new Text("User Profile", "H1"), 0, 0, Alignment.LEFT);

    // Row 1: A label and an input field
    root.add(new Text("Full Name:", "Label"), 1, 0, Alignment.LEFT);
    root.add(new TextBox(), 1, 1, Alignment.FILL); // Fills the whole column!

    // Row 2: An icon and some body text
    root.add(new Icon("user-circle"), 2, 0, Alignment.CENTER);
    root.add(new Text("Standard Account", "Body"), 2, 1, Alignment.LEFT);

    // Row 3: A button aligned to the right
    Button saveBtn = new Button("Save Changes");
    root.add(saveBtn, 3, 1, Alignment.RIGHT);
}

What’s happening here?

  • Coordinate (1, 1): The TextBox is using Alignment.FILL, making it look like a professional, wide input field.
  • Coordinate (3, 1): Even though the button is in the same column as the textbox, Alignment.RIGHT keeps it tucked neatly in the corner.
  • The “Invisible” Grid: Notice how we align labels to the LEFT and inputs to the FILL. This creates a clean, vertical “spine” down the middle of your page.

Components

What is a Component?

Every UI element in ezui—whether it’s a Button, a TextBox, or an Icon—is a Component. Think of “Component” as the DNA that all widgets share.

Because of this shared DNA, you can interact with any widget using the same simple methods.

The Interaction Superpowers:

  1. onClick(Runnable action): Runs code when the user clicks the component.
  2. onHover(Runnable action): Runs code when the user’s mouse moves over the component.

Example: Making an Icon Interactive

Usually, icons are just for decoration. But in ezui, because an Icon is a Component, you can make it act like a button!

Icon helpIcon = new Icon("question-circle");

// Add a click action to an icon!
helpIcon.onClick(() -> {
    System.out.println("User needs help!");
});

// Add a hover effect
helpIcon.onHover(() -> {
    helpIcon.setColor("Primary"); // Change color when mouse is over it
});

root.add(helpIcon, 0, 1, Alignment.RIGHT);

Why This Matters:

  • Consistency: You don’t have to remember if a widget uses an ActionListener or a MouseListener. You just use onClick.
  • Flexibility: Want a piece of Text to be clickable like a link? Just call text.onClick(...).
  • Uniformity: Every component handles its own margins, padding, and basic styling, so your app always looks “correct” regardless of which widget you use.

The Runnable Interface: “A Task in a Box”

What is a Task?

Usually, when you write System.out.println("Hello"), it runs immediately. But in a UI, we need to save that code and run it later — only when a user clicks a button.

In other words, we need a way to package that code into a “task” that can be executed on demand.

To do this, Java uses the Runnable interface. Think of it as a “Contract” that says: “I promise to provide a run() method.”

The Formal Way (The Class)

You create a separate class that “implements” the task you want to perform.

// 1. Define the Task
public class PrintTask implements Runnable {
    @Override
    public void run() {
        System.out.println("The button was clicked!");
    }
}

// 2. Use it in your Page
PrintTask myTask = new PrintTask();
myButton.onClick(myTask); // Give the "task" to the button

Introducing Lambdas: “The Professional Shortcut”

Why use a Shortcut?

Creating a whole new file/class just to print one line of text is a lot of work. Java created Lambdas as a way to write that run() method directly inside your onClick.

How the Transformation Happens:

The computer is smart. It knows onClick needs a Runnable. It knows Runnable only has one method (run). So, we can delete all the “labels” and just keep the “logic.”

From Class to Lambda:

  1. Start: public class MyTask implements Runnable { public void run() { System.out.println("Hi"); }}
  2. Remove the header: () -> { System.out.println("Hi"); }

The Lambda Syntax:

  • () : The empty parentheses represent the run() method’s parameters.
  • -> : The “Arrow” points to the code you want to execute.
  • { } : The “Body” contains your instructions.
// The ezui way! No extra classes needed.
myButton.onClick(() -> {
    System.out.println("This runs inside a Lambda!");
});

Lambda Rules of Thumb

When to use which?

  • Use a Lambda for short, simple tasks (like opening a new page or printing a message).
  • Use a Formal Class if your task is 50+ lines of code or needs to be reused in 10 different places.

The “Brace” Rule:

  • One line of code? You can skip the { }.
btn.onClick(() -> System.out.println("Fast!"));
  • Multiple lines of code? You must use { }.
btn.onClick(() -> {
    System.out.println("Line 1");
    System.out.println("Line 2");
});

🟧 SECTION 2: BASIC INPUTS & WIDGETS

Button: The Main Action Tool

What is a Button?

A Button is the primary way users trigger events in your application. In ezui, buttons are designed to look modern and professional by default.

The Three Constructors

You can create a button in three different ways depending on your UI needs:

  1. Text Only: new Button("Submit")
  2. Icon Only: new Button("play") (Uses FontAwesome)
  3. Text + Icon: new Button("Delete", "trash")
// Example: A professional "Save" button
Button saveBtn = new Button("Save Changes", "save");

// Add the logic using a Lambda!
saveBtn.onClick(() -> {
    System.out.println("Saving to database...");
    // You could also call a method here: saveUserData();
});

root.add(saveBtn, 2, 1, Alignment.RIGHT);

Pro-Tips:

  • Icon Search: You can find all the valid icon strings (like "user", "check", "envelope") by searching the FontAwesomeSolid library.
  • Consistency: Use Alignment.FILL if you want a column of buttons to all be the same width (like a vertical menu).
  • Feedback: Always provide feedback in your onClick. If the button doesn’t print something or change the screen, the user might think the app is frozen!

DropDownButton: The “Action Menu”

What is a DropDownButton?

Think of this as a Folder for Actions. When the user clicks it, a menu pops down with a list of choices. Each choice has its own Lambda (task) attached to it.

How to Build a Menu:

  1. Create the Button: Give it a main label (e.g., “Actions” or “Settings”).
  2. Add Options: Use the .addOption() method for every choice you want to provide.
// 1. Create the base button
DropDownButton menuBtn = new DropDownButton("File Options", "folder-open");

// 2. Add options with their own Lambdas
menuBtn.addOption("Download PDF", () -> {
    System.out.println("Generating PDF...");
});

menuBtn.addOption("Export to CSV", () -> {
    System.out.println("Exporting data...");
});

// 3. Add a "Danger" action
menuBtn.addOption("Delete Record", () -> {
    System.out.println("Deleting...");
});

root.add(menuBtn, 0, 1, Alignment.RIGHT);

When to use a DropDownButton:

  • Toolbar Actions: When you have “File”, “Edit”, or “View” menus.
  • Filtering: “Sort by Name”, “Sort by Date”, “Sort by Grade”.
  • Space Saving: If you have 4 buttons but only 1 grid cell left, use a DropDownButton!

Tip:

“A regular Button is for one main action. A DropDownButton is for a list of related actions. It keeps your app looking organized instead of crowded.”

Text: Labels & Headings

Use Text for any non-editable labels.

  • Styles: "H1", "H2", "Label", "Body".
Text title = new Text("Welcome Back", "H1");

TextBox: Single-Line Input

Standard input field. Now supports password masking!

TextBox userField = new TextBox();
TextBox passField = new TextBox();
passField.setMasked(true); // Turns input into dots/stars

String input = userField.getText();

TextArea: Multi-Line Input

For longer content like notes or descriptions.

TextArea bio = new TextArea();
bio.setText("Line 1\nLine 2");

Icon: Visual Symbols

Uses the FontAwesomeSolid set.

  • Case-insensitive codes (e.g., "home", "user-circle", "arrow-left").

  • Invalid codes automatically default to a ? icon.

Icon gear = new Icon("cog");
gear.setSize(32);
gear.setColor("Primary");

🟩 SECTION 4: DATA & ADVANCED

ITable: The Data Contract

To show objects in a Table, your class must implement ITable.

public class Student implements ITable {
    private String name;
    private double gpa;

    public String[] getHeaders() { return new String[]{"Name", "GPA"}; }
    public Object[] getValues() { return new Object[]{name, gpa}; }
}

Table: Dynamic Data Display

The Table stays in sync with a List.

  • Call refresh() after modifying the list to update the UI.

  • Displays “No Data Available” if the list is empty.

List<Student> students = new ArrayList<>();
Table<Student> table = new Table<>(students);

students.add(new Student("Alice", 4.0));
table.refresh(); // Table now builds its columns and rows

Chart: Visual Data

Turn Map data into Bar or Pie charts instantly.

Map<String, Double> data = Map.of("Passing", 85.0, "Failing", 15.0);
Chart pie = Chart.createPieChart("Class Stats", data);

Alignment & ColumnStyle

Control your layout precisely.

  • Alignment: LEFT, CENTER, RIGHT, FILL.

  • ColumnStyle:

    • ColumnStyle.GROW(): Takes up all extra space.

    • ColumnStyle.FIXED(): Only takes what it needs.

    • ColumnStyle.WIDTH(px): Forces a specific width.
