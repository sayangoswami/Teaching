ezui: The Modern Java UI Toolkit

🟦 SECTION 1: THE STRUCTURE

What is ezui?

ezui is a beginner-friendly Java library designed to bridge the gap between “Learning Java” and “Building Apps.”

  • Why use it? No messy LayoutManagers, built-in navigation, and modern styling out of the box.

  • The Package: Always start with import ezui.*;

Big-Picture Architecture

  • Window: The “House” (Main Frame + Sidebar).

  • Page: A “Room” (One specific screen).

  • Grid: The “Furniture Layout” (Rows and Columns).

  • Component: The “Furniture” (Buttons, Text, etc.).

Window: The App Shell

The House for Your Pages

The Window is the “Master Container.” It creates a modern, split-pane interface with a Navigation Sidebar on the left and a Scrollable Content Area on the right.

The 4-Step Setup

To launch your application, follow this exact sequence in your Main.java The program wont compile yet, because we haven’t defined the Login and Dashboard pages, but this is the skeleton you will build on top of.


// 1. Set the Style (Do this FIRST!)
Theme.initialize("Light", "Modern");

// 2. Create the Window (Title, Width, Height)
Window app = new Window("Student Portal", 1000, 700);

// 3. Register your Pages 
// NOTE: The FIRST page you add will be the one displayed at startup!
app.addPage(new LoginPage()); 
app.addPage(new DashboardPage());

// 4. Launch!
app.show();

Tips:

  • Sidebar Magic: Every time you call addPage(), a button with that Page’s name is automatically created in the sidebar. Clicking it switches the screen instantly.
  • Smart Sizing: Use 1000x700 as a safe default. It fits perfectly on most laptops without hiding the taskbar.
  • The “Show” Rule: Always call app.show() or app.setVisible(true) at the very end of your main method to ensure all your components are properly sized and ready to view.

Page — Building Your Screen

The Blueprint of a Screen

Every screen in your app (like Login, Dashboard, or Settings) must extend a Page. Think of a Page as a named container that holds a Grid. We will talk about Grids in the next slide.

The Page Constructor

When you create a Page, you must tell ezui three things: its Name, how many Rows it has, and how many Columns it has.

public class DashboardPage extends Page {
    public DashboardPage() {
        // super(Name, Rows, Columns, ColumnStyles...)
        super("Dashboard", 3, 2, ColumnStyle.FIXED(), ColumnStyle.GROW());
    }

    @Override
    public void onCreate() {
        // This is where you add your components!
    }
}

Understanding the Arguments:

  1. Name ("Dashboard"): This is the text that will appear on the button in the Sidebar.
  2. Rows (3): How many horizontal “slots” your page has from top to bottom.
  3. Columns (2): How many vertical “slots” your page has from left to right.
  4. ColumnStyles: These tell the columns how to behave.
    • ColumnStyle.FIXED(): Stays as small as possible (good for labels/icons).
    • ColumnStyle.GROW(): Stretches to fill all remaining screen space (good for Tables/Charts).
    • ColumnStyle.WIDTH(px): Stays fixed at a specific width in pixels (good for side panels).

Tips:

  • The Grid is Your Canvas: You can’t add a component to a Page without a Grid slot. If you need more space, increase the Row/Column numbers in the constructor.
  • Infinite Height: Don’t worry about making the page too long! ezui automatically adds a Scrollbar if your content exceeds the window height.
  • The onCreate Rule: Always put your root.add() code inside onCreate. If you put it in the constructor, the Window might not be ready to display it yet!

Grid: The Layout Engine

Everything inside a Page lives in a Grid.

The “Map” of Your Screen

Think of a Grid as a spreadsheet. Every “cell” has a unique address: (row, column). In programming, we always start counting at 0.

How to Add Components You use the root.add() method inside your Page’s onCreate() function.

// 1. Simple placement (Fills the column, aligns to top)
root.add(new Text("Header"), 0, 0);

// 2. Precise placement (Choose your alignment)
root.add(new Button("Save"), 1, 1, Alignment.RIGHT);

The Alignment Rulebook:

  • Alignment.LEFT: Sticks the component to the left side of the cell.
  • Alignment.CENTER: Places the component right in the middle.
  • Alignment.RIGHT: Sticks the component to the right side.
  • Alignment.FILL: Stretches the component to be as wide as the entire column (Perfect for TextBoxes and Buttons!).

Tips:

  • Overlapping: You cannot put two components in the same (row, column) address. The second one will simply replace the first!
  • Empty Spaces: You don’t have to fill every cell. If you leave (0, 1) empty, it will just show as a blank space in your layout.
  • Pro-Tip: Use Alignment.FILL for your input fields (TextBox) so they look consistent and professional across the page.

The onCreate Master Example

Putting the Grid to Work

Inside your Page class, the onCreate() method is your construction site. Here is how you would build a simple “Profile” layout using the Grid coordinates we just learned.

@Override
public void onCreate() {
    // Row 0: A big heading that spans the top
    root.add(new Text("User Profile", "H1"), 0, 0, Alignment.LEFT);

    // Row 1: A label and an input field
    root.add(new Text("Full Name:", "Label"), 1, 0, Alignment.LEFT);
    root.add(new TextBox(), 1, 1, Alignment.FILL); // Fills the whole column!

    // Row 2: An icon and some body text
    root.add(new Icon("user-circle"), 2, 0, Alignment.CENTER);
    root.add(new Text("Standard Account", "Body"), 2, 1, Alignment.LEFT);

    // Row 3: A button aligned to the right
    Button saveBtn = new Button("Save Changes");
    root.add(saveBtn, 3, 1, Alignment.RIGHT);
}

What’s happening here?

  • Coordinate (1, 1): The TextBox is using Alignment.FILL, making it look like a professional, wide input field.
  • Coordinate (3, 1): Even though the button is in the same column as the textbox, Alignment.RIGHT keeps it tucked neatly in the corner.
  • The “Invisible” Grid: Notice how we align labels to the LEFT and inputs to the FILL. This creates a clean, vertical “spine” down the middle of your page.

Components

What is a Component?

Every UI element in ezui—whether it’s a Button, a TextBox, or an Icon—is a Component. Think of “Component” as the DNA that all widgets share.

Because of this shared DNA, you can interact with any widget using the same simple methods.

The Interaction Superpowers:

  1. onClick(Runnable action): Runs code when the user clicks the component.
  2. onHover(Runnable action): Runs code when the user’s mouse moves over the component.

Example: Making an Icon Interactive

Usually, icons are just for decoration. But in ezui, because an Icon is a Component, you can make it act like a button!

Icon helpIcon = new Icon("question-circle");

// Add a click action to an icon!
helpIcon.onClick(() -> {
    System.out.println("User needs help!");
});

// Add a hover effect
helpIcon.onHover(() -> {
    helpIcon.setColor("Primary"); // Change color when mouse is over it
});

root.add(helpIcon, 0, 1, Alignment.RIGHT);

Why This Matters:

  • Consistency: You don’t have to remember if a widget uses an ActionListener or a MouseListener. You just use onClick.
  • Flexibility: Want a piece of Text to be clickable like a link? Just call text.onClick(...).
  • Uniformity: Every component handles its own margins, padding, and basic styling, so your app always looks “correct” regardless of which widget you use.

The Runnable Interface: “A Task in a Box”

What is a Task?

Usually, when you write System.out.println("Hello"), it runs immediately. But in a UI, we need to save that code and run it later — only when a user clicks a button.

In other words, we need a way to package that code into a “task” that can be executed on demand.

To do this, Java uses the Runnable interface. Think of it as a “Contract” that says: “I promise to provide a run() method.”

The Formal Way (The Class)

You create a separate class that “implements” the task you want to perform.

// 1. Define the Task
public class PrintTask implements Runnable {
    @Override
    public void run() {
        System.out.println("The button was clicked!");
    }
}

// 2. Use it in your Page
PrintTask myTask = new PrintTask();
myButton.onClick(myTask); // Give the "task" to the button

Introducing Lambdas: “The Professional Shortcut”

Why use a Shortcut?

Creating a whole new file/class just to print one line of text is a lot of work. Java created Lambdas as a way to write that run() method directly inside your onClick.

How the Transformation Happens:

The computer is smart. It knows onClick needs a Runnable. It knows Runnable only has one method (run). So, we can delete all the “labels” and just keep the “logic.”

From Class to Lambda:

  1. Class:
public class MyTask implements Runnable {
    public void run() {
        System.out.println("Button clicked!");
    }
}
// in your button setup
myButton.onClick(new MyTask());
  1. Lambda:
myButton.onClick(() -> {
    System.out.println("Button clicked!");
});

The Lambda Syntax:

myButton.onClick(() -> {
    System.out.println("This runs inside a Lambda!");
});
  • () : The empty parentheses represent the run() method’s parameters.
  • -> : The “Arrow” points to the code you want to execute.
  • { } : The “Body” contains your instructions.

Lambda Rules of Thumb

When to use which?

  • Use a Lambda for short, simple tasks (like opening a new page or printing a message).
  • Use a Formal Class if your task is 50+ lines of code or needs to be reused in 10 different places.

The “Brace” Rule:

  • One line of code? You can skip the { }.
btn.onClick(() -> System.out.println("Fast!"));
  • Multiple lines of code? You must use { }.
btn.onClick(() -> {
    System.out.println("Line 1");
    System.out.println("Line 2");
});

Lambda Scope: The “Look, But Don’t Touch” Rule

The Concept of Capture

When you write a Lambda, it “captures” the variables in the code surrounding it. However, Java has a strict rule: a Lambda can only use variables that are Effectively Final.

What is “Effectively Final”?

It means a variable whose value never changes after it is initialized. You don’t have to type the word final, but you must treat it as if it were constant.

The Mistake (Variable Changes)

int counter = 0;
Button btn = new Button("Click Me");

btn.onClick(() -> {
    // ERROR: Local variable counter defined in an enclosing scope must be final
    // or effectively final
    counter++; 
});

counter = 5; // Changing this later also breaks the Lambda above!

The Correct Way (Using Objects) If you need to update a value, you should update a Component or an Object, not a primitive int or String.

Text status = new Text("Count: 0", "Body");
int[] counter = {0}; // Arrays are objects, so the reference stays final!

btn.onClick(() -> {
    counter[0]++; // This works! We are changing the content, not the array itself.
    status.setText("Count: " + counter[0]);
});

Tips:

  • Think of it as a Snapshot: When the Lambda is created, it takes a “picture” of the variables around it. If you move or change those variables later, the “picture” becomes invalid.
  • The ezui Solution: Usually, you don’t need to change variables. You just want to change the UI. Use setText(), setColor(), or refresh() inside your Lambdas—those methods are always safe to call!

🟧 SECTION 2: BASIC INPUTS & WIDGETS

Button: The Main Action Tool

What is a Button?

A Button is the primary way users trigger events in your application. In ezui, buttons are designed to look modern and professional by default.

The Three Constructors

You can create a button in three different ways depending on your UI needs:

  1. Text Only: new Button("Submit")
  2. Icon Only: new Button("play") (Uses FontAwesome)
  3. Text + Icon: new Button("Delete", "trash")
// Example: A professional "Save" button
Button saveBtn = new Button("Save Changes", "save");

// Add the logic using a Lambda!
saveBtn.onClick(() -> {
    System.out.println("Saving to database...");
    // You could also call a method here: saveUserData();
});

root.add(saveBtn, 2, 1, Alignment.RIGHT);

Pro-Tips:

  • Icon Search: You can find all the valid icon strings (like "user", "check", "envelope") by searching the FontAwesomeSolid library.
  • Consistency: Use Alignment.FILL if you want a column of buttons to all be the same width (like a vertical menu).
  • Feedback: Always provide feedback in your onClick. If the button doesn’t print something or change the screen, the user might think the app is frozen!

DropDownButton: The “Action Menu”

What is a DropDownButton?

Think of this as a Folder for Actions. When the user clicks it, a menu pops down with a list of choices. Each choice has its own Lambda (task) attached to it.

How to Build a Menu:

  1. Create the Button: Give it a main label (e.g., “Actions” or “Settings”).
  2. Add Options: Use the .addOption() method for every choice you want to provide.
// 1. Create the base button
DropDownButton menuBtn = new DropDownButton("File Options", "folder-open");

// 2. Add options with their own Lambdas
menuBtn.addOption("Download PDF", () -> {
    System.out.println("Generating PDF...");
});

menuBtn.addOption("Export to CSV", () -> {
    System.out.println("Exporting data...");
});

// 3. Add a "Danger" action
menuBtn.addOption("Delete Record", () -> {
    System.out.println("Deleting...");
});

root.add(menuBtn, 0, 1, Alignment.RIGHT);

When to use a DropDownButton:

  • Toolbar Actions: When you have “File”, “Edit”, or “View” menus.
  • Filtering: “Sort by Name”, “Sort by Date”, “Sort by Grade”.
  • Space Saving: If you have 4 buttons but only 1 grid cell left, use a DropDownButton!

Tip:

“A regular Button is for one main action. A DropDownButton is for a list of related actions. It keeps your app looking organized instead of crowded.”

Text: Typography & Styles

What is the Text Component? In ezui, you don’t just “draw strings.” You use the Text component to create labels, headings, and descriptions. Each Text object follows the rules of your chosen Theme.

The Style Cheat Sheet

When you create a Text object, you can pass a Style String as the second argument. This tells ezui exactly how to format the font size, weight, and spacing.

Style Name Purpose Visual Look
"H1" Main Page Titles Large, Bold, High Contrast
"H2" Section Headers Medium-Large, Bold
"Label" Input Descriptions Small, Bold (Perfect for “Name:”, etc.)
"Body" General Paragraphs Standard Size, Normal Weight

Text: Typography & Styles

Code Example:

// Create a big title
Text title = new Text("Student Dashboard", "H1");
root.add(title, 0, 0, Alignment.LEFT);

// Create a small label for a field
Text nameLabel = new Text("Full Name:", "Label");
root.add(nameLabel, 1, 0, Alignment.LEFT);

Text: Typography & Styles

Changing Content Dynamically

Since Text is a Component, you can change what it says inside a Lambda!

Text status = new Text("System Ready", "Body");

myBtn.onClick(() -> {
    status.setText("Processing...");
});

Tips:

  • Consistency is Key: Don’t use "H1" for everything. Use one "H1" per page, and use "Label" for all your form inputs so they line up perfectly.
  • The “Alignment” Pair: Usually, you want to pair a Text (Alignment.LEFT) with a TextBox (Alignment.FILL) in the same row to create a clean form layout.

TextBox: Capturing User Input

What is a TextBox? A TextBox is a single-line input field. Unlike Text, which is for display only, a TextBox allows the user to type.

The “Get and Set” Pattern To interact with a TextBox, you will primarily use two methods: 1. getText(): Pulls the current string out of the box (usually inside an onClick Lambda). 2. setText(String): Replaces the text inside the box programmatically.

TextBox: Capturing User Input

Code Example: A Simple Greeting

TextBox nameField = new TextBox();
Button greetBtn = new Button("Say Hello");

greetBtn.onClick(() -> {
    // 1. Grab the input
    String input = nameField.getText();
    
    // 2. Use it!
    System.out.println("Hello, " + input + "!");
    
    // 3. Optional: Clear the box for the next entry
    nameField.setText(""); 
});

root.add(nameField, 0, 0, Alignment.FILL);
root.add(greetBtn, 0, 1, Alignment.RIGHT);

TextBox: Capturing User Input

Feature: Masking Passwords

If you are building a login screen, you don’t want the password visible. Use setMasked(true) to turn the characters into dots.

TextBox passField = new TextBox();
passField.setMasked(true); // Secure input!

Tips:

  • Alignment.FILL is your friend: TextBoxes look best when they stretch to fill their column.
  • Empty Inputs: Always check if getText().isEmpty() before processing data. If the user clicks “Submit” without typing, your code might crash or behave strangely!
  • Scope Reminder: Remember to define your TextBox outside the Lambda (as a variable in onCreate) so the Lambda can “see” it and call getText().

TextArea: Handling “Big” Strings

What is a TextArea?

While a TextBox is a thin strip for a single line of text, a TextArea is a large, scrollable box. Use this when you need the user to provide a Description, a Bio, or Notes.

Key Differences from TextBox:

  1. Multi-line: It automatically wraps text to the next line.
  2. Scrollable: If the user types a whole essay, they can scroll up and down within the box.
  3. Height: In your root.add(), a TextArea usually needs more “Vertical Room” to look right.

TextArea: Handling “Big” Strings

Code Example: A Feedback Form

// Create a large area for comments
TextArea feedbackBox = new TextArea();

// You can set a prompt to help the user
feedbackBox.setPlaceholder("Enter your suggestions here...");

Button sendBtn = new Button("Submit Feedback", "paper-plane");

sendBtn.onClick(() -> {
    String longMessage = feedbackBox.getText();
    System.out.println("User sent: " + longMessage);
    
    // Clear it after sending
    feedbackBox.setText("");
});

root.add(feedbackBox, 1, 0, Alignment.FILL);
root.add(sendBtn, 2, 0, Alignment.RIGHT);

TextArea: Handling “Big” Strings

The “Alignment.FILL” Rule:

Just like the TextBox, a TextArea looks best when it uses Alignment.FILL. This allows it to expand and fill the entire grid cell, giving the user plenty of room to type.

Tips:

  • Don’t Overcrowd: Because a TextArea is tall, try to give it its own row in your Grid.
  • Reading Data: Just like the TextBox, you use .getText() to grab everything the user typed as one big String.
  • Read-Only Mode: You can use textArea.setEditable(false) to turn it into a scrollable “Terms and Conditions” or “Log” box that the user can read but not change.

Icons: Visual Shorthand

What is the Icon Component? In ezui, we use FontAwesomeSolid icons (Version 5). Instead of loading image files, you just use a “Key” string. Icons make your buttons and labels instantly recognizable.

How to use them: You can use an Icon as a standalone component or as a decoration for a Button.

// A standalone large icon for a header
Icon userIcon = new Icon("account_circle", 48); // Name and Size

// A button with an icon
Button saveBtn = new Button("Save Changes", "save"); 

root.add(userIcon, 0, 0, Alignment.CENTER);

Finding the Right Icon:

To find the correct icon name, go to the FontAwesomeSolid V5 Icons Search page and search for the icon you want. Click on it, and you’ll see the exact name you need to use in your code (e.g., “user-circle”, “save”, “trash”).

ListBox & MultiListBox: Asking user choices

What is a ListBox? Its a component that displays a list of items. The user can select one item from the list.

The MultiListBox Advantage The MultiListBox is also like a ListBox that allows users to select multiple items at once (using Ctrl+Click or Shift+Click). This is essential for things like “Assigning Tags” or “Selecting multiple students for an email.”

Notes:

  • The addItems() method in both ListBox and MultiListBox allows you to quickly populate the list with an array of strings. You can also add items one by one using addItem().
  • The SelectedItem method in ListBox returns the currently selected item from the list, or null if nothing is selected. For the MultiListBox, getSelectedItems() returns a list of all selected items, which can be empty if no selections are made.

ListBox & MultiListBox: Asking user choices

Code Example: A Resource Manager

// 1. Single Selection
ListBox<String> categoryList = new ListBox<>();
categoryList.addItems("Homework", "Quizzes", "Exams");

// 2. Multiple Selection
MultiListBox<String> studentList = new MultiListBox<>();
studentList.addItems("Alice", "Bob", "Charlie", "David");

// 3. Logic: Get what was selected
Button processBtn = new Button("Process Selected");
processBtn.onClick(() -> {
    List<String> selectedStudents = studentList.getSelectedItems();
    String selectedCategory = categoryList.getSelectedItem();
    System.out.println("Processing " + selectedStudents.size() + " students in category: " + selectedCategory);
});

root.add(categoryList, 1, 0, Alignment.FILL);
root.add(studentList, 1, 1, Alignment.FILL);

ComboBox: The Dropdown ListBox

What is a ComboBox? Use a ComboBox when you have a list of options (like “Select a Major” or “Pick a Color”) but you want to save space on the screen. It only shows the full list when clicked.

The Code Pattern:

ComboBox<String> majorPicker = new ComboBox<>();
majorPicker.addItem("Computer Science");
majorPicker.addItem("Mathematics");
majorPicker.addItem("Physics");

// Get the choice
String selected = majorPicker.getSelectedItem();

root.add(majorPicker, 1, 1, Alignment.FILL);

Table: Automatic Data Mapping

What makes this Table special?

In ezui, the Table is “Smart.” You don’t have to tell it what the headers are or manually type every row. Instead, you give it a List of Objects, and the Table handles the rest.

Step 1: The ITable Contract

To show your data in a table, your class (like Student or Book) must “implement” the ITable interface. This tells the table how to “read” your object.

public class Student implements ITable {
    private String id, name, grade;

    // The Table calls this to build the top header row
    public String[] getHeaders() { 
        return new String[]{"ID", "Student Name", "Current Grade"}; 
    }

    // The Table calls this to fill in each row
    public String[] getValues() { 
        return new String[]{id, name, grade}; 
    }
}

Table: Automatic Data Mapping

What makes this Table special?

In ezui, the Table is “Smart.” You don’t have to tell it what the headers are or manually type every row. Instead, you give it a List of Objects, and the Table handles the rest.

Step 2: Connecting the Data

Once your objects follow the rules, creating the table is a single line of code!

// 1. Get your data (usually from a Database or List)
ArrayList<Student> myClass = database.getStudents();

// 2. Create the table and pass the list
Table<Student> studentTable = new Table<>(myClass);

root.add(studentTable, 0, 0, Alignment.FILL);

Selecting & Refreshing Data

Grabbing the Selected Object

Because this table uses Generics (<Student>), when you ask for the “Selected Object,” it doesn’t give you a row of text—it gives you the actual Student object.

Button editBtn = new Button("Edit Student", "user-edit");

editBtn.onClick(() -> {
    // Grab the actual Student object from the clicked row
    Student selected = studentTable.getSelectedObject();

    if (selected != null) {
        System.out.println("Editing: " + selected.getName());
        // Logic to open an edit page...
    }
});

The refresh() Method

If you add a new student to your ArrayList, the table won’t know… unless you tell it.

myClass.add(new Student("105", "Charlie", "A-"));
studentTable.refresh(); // The UI updates instantly!

Why this is “Industry Standard” Design

  • Type Safety: The table knows exactly what kind of objects it is holding (<Student>, <Book>, etc.).
  • Less Code: You write the headers and values once in the object class, and the table works everywhere.
  • Smart UI: If the list is empty, the table automatically displays a “No Data Available” message.

Tip:

“Always remember: The Table is a Mirror. If you want to change what the table shows, change the data in your ArrayList and then call .refresh().”

The “Selection” Logic

All three of these (ComboBox, ListBox, MultiListBox) share a common behavior: they trigger an event when the user changes their mind.

majorPicker.onSelectionChange(() -> {
    System.out.println("User switched to: " + majorPicker.getSelectedItem());
});

Tips:

  • Generics Matter: Just like your Table<T>, these components are typed. If you are picking Students, use ComboBox<Student>.

  • The “Zero” State: Always check if getSelectedItem() is null before using it. If the user hasn’t clicked anything yet, your code will crash if you don’t check!

  • Space Management: Use a ComboBox for 5+ items to save space. Use a ListBox for 2-4 items so the user can see them immediately.

Charts: Visualizing the Data

Why use Charts? Numbers in a Table are great, but a Chart tells a story instantly. In ezui, Charts are created using Static Factory Methods. This means you don’t use new Chart(), you ask the Chart class to build one for you.

1. The Bar Chart

Best for comparing different categories (e.g., “Average Grade per Subject”).

Map<String, Double> grades = new HashMap<>();
grades.put("Math", 85.0);
grades.put("History", 92.0);
grades.put("Science", 78.0);

// Title, X-Axis Label, Y-Axis Label, Data
Chart bar = Chart.createBarChart("Class Performance", "Subjects", "Average %", grades);

root.add(bar, 0, 0, Alignment.FILL);

Charts: Visualizing the Data

Why use Charts? Numbers in a Table are great, but a Chart tells a story instantly. In ezui, Charts are created using Static Factory Methods. This means you don’t use new Chart(), you ask the Chart class to build one for you.

2. The Pie Chart

Best for showing parts of a whole (e.g., “Attendance Status” or “Grade Distribution”).

Map<String, Double> attendance = new HashMap<>();
attendance.put("Present", 22.0);
attendance.put("Absent", 3.0);
attendance.put("Excused", 1.0);

// Title, Data
Chart pie = Chart.createPieChart("Today's Attendance", attendance);

root.add(pie, 0, 1, Alignment.FILL);

  1. Initialize Theme: Theme.initialize("Light", "Modern");

  2. Setup Window: Create your Window.

  3. Build Pages: Create your Page classes and use root.add.

  4. Logic: Use onClick to change data and table.refresh() to show it.
