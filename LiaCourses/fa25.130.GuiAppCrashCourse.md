# GUI App Crash Course


## What is This Library?

This library is a set of pre-built Java classes that let you build real, interactive applications _without_ worrying about all the complex graphics code.

**Our Goal:** Use this library to build apps and see **Object-Oriented Programming in action.**

We will focus on two key OOP concepts:

1. **Interfaces** (The "IS-A" Contract)
    
2. **Composition** (The "HAS-A" Building Block)
    

## The Big Picture: Our Main Classes

You will build your _entire_ app using just a few components:

- **`App`**: The main application window.
    
- **`Page`**: A single tab in your application.
    
- **`Text`**: A component for displaying text.
    
- **`Form`**: A component for user input.
    
- **`Table`**: A component for displaying data in a grid.
    
- **`Chart`**: A component for visualizing data (pie/bar).
    
- **`StatCard`**: A component for showing a single key number.
    
- **`Panel`**: A component that _contains_ other components.
    

## How to use: `App` and `Page`

- The **`App`** is the main window. You only create **one**.
    
- A **`Page`** is a tab that you add _to_ the `App`.
    
- A `Page` can hold _one_ component (like a `Table`) or a `Panel` (which can hold many).
    

```java
// 1. Create the main application window
App myApp = new App("My First App", 800, 600);

// 2. Create a page (a tab)
Page mainPage = new Page();

// 3. Add the page to the app
myApp.addPage("Dashboard", mainPage);

// (We'll add components to the page next...)

// 4. Show the app
myApp.show();
```

## How to use: `Text`

- The simplest component. It just shows text.
    
- It's great for instructions, descriptions, or reports.
    

```java
// Create a text component
Text welcomeText = new Text("Welcome to my app!");

// Add a title and more text
welcomeText.setTitle("Instructions");
welcomeText.addText("Please fill out the form on the next page.");

// Add the text component to our page
mainPage.addComponent(welcomeText);
```

(This assumes `mainPage` was created like on the last slide)

## How to use: `Form`

- The `Form` component is built for user input.
    
- You give it a list of field names, and it builds the form.
    
- It _requires_ a **`FormSubmitHandler`** object to tell it what to do when the "Submit" button is clicked.
    

```java
// 1. Define the fields for the form
String[] formFields = {"Name", "Major", "Email"};

// 2. Create a handler object
// (This class MUST implement the FormSubmitHandler interface)
MyFormHandler handler = new MyFormHandler();

// 3. Create the form
Form studentForm = new Form("New Student", formFields, handler);

// 4. Add the form to a new page
Page formPage = new Page();
formPage.addComponent(studentForm);
myApp.addPage("Add Student", formPage);
```

## How to use: `StatCard`

- The `StatCard` is for showing a single, important piece of data.
    
- It's perfect for dashboards.
    

```java
// Create a StatCard
StatCard totalStudents = new StatCard("Total Students", "142");

// You can add this directly to a page,
// but it's more useful inside a Panel...
```

## How to use: `Panel` (Composition)

- The **`Panel`** is a special component that acts as a **container**.
    
- It lets you build complex layouts by _combining_ other components.
    
- This is our best example of **Composition** (a "HAS-A" relationship).
    

```java
// Create a Panel with a 2-row, 2-column grid
// (10px padding between components)
Panel dashboardPanel = new Panel(2, 2, 10, 10);

// Create components to put inside it
StatCard totalStudents = new StatCard("Total Students", "142");
StatCard avgGpa = new StatCard("Average GPA", "3.4");
Chart majorChart = new Chart(...);
Chart yearChart = new Chart(...);

// Add components to the Panel
dashboardPanel.addComponent(totalStudents);
dashboardPanel.addComponent(avgGpa);
dashboardPanel.addComponent(majorChart);
dashboardPanel.addComponent(yearChart);

// Add the *single* Panel to your page
mainPage.addComponent(dashboardPanel);
```

## How to use: `Table` (Data)

- The `Table` holds a list of your data objects.
    
- It is the most complex and powerful component.
    

### Creating a Table:

```java
// 1. Define the column headers
String[] studentColumns = {"ID", "Name", "Major"};

// 2. Create your data list (must be an ArrayList)
ArrayList students = new ArrayList();

// 3. Add your data objects (must implement Rowable)
students.add(new Student("s1", "Alice", "CS"));
students.add(new Student("s2", "Bob", "Math"));

// 4. Create the table
Table studentTable = new Table(studentColumns, students);

// 5. Add the table to a page
Page studentPage = new Page();
studentPage.addComponent(studentTable);
myApp.addPage("Students", studentPage);
```


The constructor takes a `String[]` for headers and an `ArrayList` of objects that implement the `Rowable` interface.

## How to use: `Table` (Updating)

The `Table` class has methods to keep your data in sync.

### Adding a Single Row:

Use this when your `Form` submits a new object.

```java
// Assume 'newStudent' is a Rowable object from your form
Student newStudent = new Student("s3", "Charlie", "Art");

// This adds to BOTH the internal list AND the visible table
studentTable.addRow(newStudent);
```

### Refreshing the Whole Table:

Use this if you modify your original `ArrayList` _externally_ (e.g., sorting it, removing items) and need the table to match.

```java
// External change to your list
students.remove(0); // Remove "Alice"

// Tell the table to update itself from the list
studentTable.refresh();
```

## How to use: `Chart`

- The `Chart` component can draw a Pie or Bar chart.
    
- It takes a title, a chart type, and lists of labels and values.
    
- The values **must** be `Number` objects (like `Integer` or `Double`).
    

```java
// 1. Define the data
String[] keys = {"CS", "Math", "Art", "Biology"};
Number[] values = {new Integer(45), 
                   new Integer(20), 
                   new Integer(15), 
                   new Integer(30)};

// 2. Create the chart
Chart majorChart = new Chart(
    "Students by Major", 
    "pie",  // or "bar"
    keys, 
    values
);

// 3. Add it to a page or panel
dashboardPanel.addComponent(majorChart);
```

## The Big Connection: OOP in Practice

Why is this setup so "Object-Oriented"?

### 1. `Number` is an **Abstract Class**

- The `Chart` constructor asks for an array of `Number`.
    
- You can pass it `Integer` or `Double` objects.
    
- Why? Because `Integer` **IS-A** `Number` and `Double` **IS-A** `Number`.
    
- The `Chart` doesn't care _what kind_ of number it is, only that it _is a_ `Number` and has a `doubleValue()` method.
    
- This is **Polymorphism**!
    

### 2. `FormSubmitHandler` is an **Interface**

- The `Form` _needs_ an object that implements `FormSubmitHandler`.
    
- It doesn't care _what_ your class is named (`MyHandler`, `StudentFormLogic`, etc.) as long as it "signs the contract" by implementing the `onSubmit` method.
    

### 3. `Rowable` is an **Interface**

- The `Table` _needs_ objects that implement `Rowable`.
    
- It doesn't care about your `Student` or `Task` object, only that it "signs the contract" by implementing the `getRowData` method.
    

## Let's Talk About `Number`...

This is a **real** abstract class built into Java.

- You **cannot** create an instance of it.
    
    - `Number n = new Number();` // ERROR!
        
- It provides a _contract_ for all "number-like" classes.
    
- `Integer`, `Double`, `Float`, `Long` all _extend_ `Number`.
    
- This lets our `Chart` work with _any_ of them!
    

```java
// The Chart class can use this one variable
// to hold any object whose class extends Number
Number anyNumber = new Integer(10);
anyNumber = new Double(3.14);

// This is why the Chart is so flexible!
double value = anyNumber.doubleValue();
```

## What about `Integer`? (Wrapper Classes)

Why do we have `Integer` when we have `int`?

- `int`, `double`, `boolean` are **primitive types**.
    
- They are _not_ objects. They are just simple values.
    
- You **cannot** put them in an `ArrayList` or pass them as an `Object`.
    

**Wrapper Classes** (`Integer`, `Double`, `Boolean`) "wrap" these primitives inside a real object, so they can be treated like any other object.

|Primitive|Wrapper Class|
|---|---|
|`int`|`Integer`|
|`double`|`Double`|
|`boolean`|`Boolean`|

This is why our `Chart` needs `Integer` or `Double`â€”it needs _objects_.

## The `FormSubmitHandler` Interface

This is your **first "contract"** to implement.

- **The Contract:** "If you want to be a `FormSubmitHandler`, you _must_ provide a method named `onSubmit` that takes an array of `String`s."
    
- **Your Class:** You create a class that _implements_ this interface.
    

```java
public class MyStudentHandler implements FormSubmitHandler {

    // You MUST provide this method
    public void onSubmit(String[] formData) {
        // formData[0] is "Name"
        // formData[1] is "Major"
        // ...
        
        // 1. Create a Student from the data
        Student s = new Student(formData[0], formData[1], ...);
        
        // 2. Add the student to the table
        // (You'll need to pass the table into your handler)
        studentTable.addRow(s);
    }
}
```

## The `Rowable` Interface

This is your **second "contract"** to implement.

- **The Contract:** "If you want to be _showable in a table_ (i.e., `Rowable`), you _must_ provide a method named `getRowData` that returns a `String[]`."
    
- **Your Data Class:** You make your `Student`, `Task`, or `Item` class implement this interface.
    

```java
// Our "Student" class "IS-A" Rowable
public class Student implements Rowable {
    
    // Your normal class fields
    private String id;
    private String name;
    private String major;
    
    // Your normal constructor
    public Student(String id, String name, String major) {
        this.id = id;
        this.name = name;
        this.major = major;
    }
    
    // You MUST provide this method
    public String[] getRowData() {
        // Return the fields in the *exact order*
        // as the table's column headers.
        String[] data = {this.id, this.name, this.major};
        return data;
    }
}
```
