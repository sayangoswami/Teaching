# Arrays: Storing Data at Scale


## The "Why": The Problem of Scale

So far, we use variables to store data. This works fine for simple problems.

```java
// Store one student's score:
int score = 95;
```

But what if we need to store scores for **5 students**?

```java
// This is tedious, but... okay.
int score1 = 88;
int score2 = 92;
int score3 = 76;
int score4 = 81;
int score5 = 95;
```

What if we need to store scores for **5,000 students**? Or what if we ask the user "how many students" and **we don't know in advance**?

We need a better way.


## The Analogy

Think about this:

- **Loops** let us repeat an **action** many times.
    
- **Arrays** let us store **data** many times.
    

An **Array** is a variable that can hold _many_ values at once.

It's a container, like a list, a row of mailboxes, or a long box with compartments.


## What is an Array?

An **array** is a fixed-size, ordered collection of elements of the **same data type**.

- **Collection:** It holds multiple values in one place.
    
- **Fixed-Size:** When you create an array, you decide its size. It cannot grow or shrink.
    
- **Ordered:** Every element has a position, called an **index**.
    
- **Same Data Type:** You can have an array of `int`s, or an array of `String`s, but you can't mix them in the _same_ array.
    


## How to Create Arrays (Syntax)

There are two main ways to create an array.

**1. The "Formal" Way (When you know the size, but not the values):**

```java
// Syntax: <type>[] <name> = new <type>[<size>];

// "Create a new array of 10 integers."
int[] scores = new int[10];

// "Create a new array of 50 Strings."
String[] names = new String[50];
```

**2. The "Literal" Way (When you know the values up front):**

```java
// Syntax: <type>[] <name> = {val1, val2, ...};

// Creates an array of 5 integers.
int[] scores = {88, 92, 76, 81, 95};

// Creates an array of 3 Strings.
String[] names = {"Alice", "Bob", "Charlie"};
```


## The "Gotcha": Arrays are 0-Indexed!

This is the most important rule in all of programming.

The "position" or **index** of an array element **starts at 0**.

- The 1st element is at index `[0]`.
    
- The 2nd element is at index `[1]`.
    
- The N-th element is at index `[N - 1]`.
    

**Visual:** A `String` array of size 5: `String[] names = new String[5];`

<!-- data-type="none" data-sortable="false" -->
|Array Index:|[0]|[1]|[2]|[3]|[4]|
|---|---|---|---|---|---|
|Value:|`null`|`null`|`null`|`null`|`null`|

The index is the **offset from the start**. The first element is 0 steps away.


## Accessing & Modifying Elements

We use the square brackets `[ ]` and the **index** to get or set a value.

```java
public class Main {
    public static void main (String[] args) {
        // Create our array
        int[] scores = {88, 92, 76};

        // --- ACCESSING (getting) a value ---
        // Get the element at index 0 (the 1st element)
        int firstScore = scores[0]; // firstScore is now 88

        // Get the element at index 2 (the 3rd element)
        int thirdScore = scores[2]; // thirdScore is now 76

        // --- MODIFYING (setting) a value ---
        // Change the value at index 1 (the 2nd element)
        scores[1] = 95; // The array is now {88, 95, 76}
    }
}
```
@LIA.java(Main)

Danger: `scores[3] = 100; // CRASH!`

This causes an `ArrayIndexOutOfBoundsException`. The valid indices are 0, 1, 2.


## The `array.length` Property

How do we know how big an array is? Every array knows its own length - every array has a `.length` property[^1].

```java
public class Main {
    public static void main (String[] args) {
        int[] scores = {88, 92, 76, 81, 95};
        String[] names = {"Alice", "Bob", "Charlie"};

        int numScores = scores.length; // numScores is 5
        int numNames = names.length;   // numNames is 3
    }
}
```
@LIA.java(Main)

Key fact: `array.length` is a property (no parentheses).

This is different from `myString.length()` (which is a method).

**Key Relationship:**

- The size of the array is `myArray.length`.
    
- The _last valid index_ is `myArray.length - 1`.
    

[^1]: This statement should tell you that an array is a *class* and not a primitive type.


## The "Aha!" Moment: Arrays + Loops

This is where it all comes together.

- Arrays store lots of data.
    
- Loops are good at doing something many times.
    

We can use a `for` loop to "iterate" (visit) every element in an array.

The `for` loop and the `array.length` property are a perfect match.


## Example 1: Printing All Elements

This is the most common pattern you will ever write.

"Loop from the first index (0) up to (but not including) the length."

```java
public class Main {
    public static void main (String[] args) {
        String[] names = {"Alice", "Bob", "Charlie", "David"};

        // i = 0, 1, 2, 3
        for (int i = 0; i < names.length; i++) {
            
            // On the first loop (i=0), print names[0]
            // On the second loop (i=1), print names[1]
            // ...
            System.out.println(names[i]);
        }
    }
}
```
@LIA.java(Main)


This loop works for an array of _any_ size.


## Example 2: The Accumulator Pattern (Sum)

Let's use a loop to process the data.

Problem: Find the sum of all scores in an array.

```java
public class Main {
    public static void main (String[] args) {
        int[] scores = {88, 92, 76, 81, 95};
        int sum = 0; // 1. Initialize accumulator

        for (int i = 0; i < scores.length; i++) {
            // 2. Accumulate
            sum = sum + scores[i];
        }

        System.out.println("The sum is: " + sum); // 4. Use the result
    }
}
```
@LIA.java(Main)




## Example 3: Finding the Maximum Value

A slightly more complex accumulator pattern.

Problem: Find the highest score in the array.

```java
public class Main {
    public static void main (String[] args) {
        int[] scores = {88, 92, 76, 81, 95};

        // 1. Initialize 'max' to the *first* element
        int max = scores[0]; 

        for (int i = 1; i < scores.length; i++) {
            // 2. Check if the current element is bigger
            if (scores[i] > max) {
                // 3. If so, update 'max'
                max = scores[i];
            }
        }

        System.out.println("The highest score is: " + max);
    }
}
```
@LIA.java(Main)




## Summary: What We Learned

- Arrays solve the "too many variables" problem.
    
- An array is a **fixed-size** container for elements of the **same type**.
    
- Arrays are **Zero-Indexed**: The first element is at `[0]`.
    
- The last valid index is `myArray.length - 1`.
    
- We use `[ ]` to access and modify elements.
    
- We use `for` loops to iterate over arrays, from `i = 0` to `i < myArray.length`.
